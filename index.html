Projects

The Crystal Guard

This is a game which my group and I created in my final game design subject at university. We worked on it over the course of a semester (about 11 weeks) and presented it to our class, teachers, and a few guests from War Gaming. The Crystal Guard is a multiplayer, fps, tower-defence computer game. We chose this genre mashup in an attempt to create something new that we would enjoy playing. We made a tutorial for the game so that players could begin playing without our guidance. I worked on the networking, the grid for pathfinding and tower placement, enemy AI, and tower placement.

Tutorial

Here you can see the beginning of the tutorial for The Crystal Guard. As you complete set actions the tutorial will progress until you have a grasp on the controls and concept of the game.

A* pathfinding and grid

In this image you can see the debugged grid on which enemy AI pathfinds and player place towers. the red cubes are where enemies cannot go (unwalkable), yellow cubes are cubes that are adjactend to unwalkable cubes. The black line shows the path that ground units are planning to take and the pink line shows the path that air units will take. Pathfinding is done using a variation on a* pathfinding.

Responsive pathfinding

This picture shows what happens to enemy pathfinding when towers block their fastest route to their end goal (the crystal). The player has placed two towers in such a way that ground enemies have no room to get through that way so they have to pathfind around a longer way. This gives the player longer to kill them and if the player wants, they could lead enemies into a "killzone" with many more turrets. Also shown in the picture is the air enemy path which is unchanged as they can just fly over the player's towers.

Tower Placement

Here you can see the player looking at the ground to choose where to put their next tower. This tower is good because it causes damage to all enemies within its range. The other tower types are "Trigun" which is a minigun type tower, and "Plasma Cannon" which is a slow firing, splash damage, projectile tower. The player is placing an electric type of tower which shocks all enemies inside its radius at a constant rate.

Duplicate Dumper

Are you always finding your iTunes library full of useless duplicate files? Do you find yourself wasting whole afternoons deleting said files? Well this app is probably not the app for you!
GitHub

Expand

This is a tiny app which I made in a few hours on a Sunday afternoon. I was prompted to make something that would delete my duplicates because my iTunes library was absolutely full of them and I didn't have any programs that would automatically delete duplicates. Because I only wanted a quick fix for my duplicates problem I did not spend any time on making my app accessible or even remotely user friendly, sorry.
The main type of duplicate that made up 90% of my problem was in a simple format where a space and a number was added to the end of the filename. I decided if could make an app that would get rid of these files i would save myself a lot of time over the years. So, I designed my app, added functionality for a few other formats that I found cropped up a fair bit, and cleaned my library.
Duplicate-dumper would be a hassle for most people to use as it requires the user to run the express server on their computer. I'm not sure if there is a way to package up such an app for easy install but I don't think there is.
If you still want to use the app you can go to the GitHub page and clone the repo. You need to install npm and node if you haven't already got them, then run npm install in a terminal while in the project directory. After everything is installed, run npm start in the same directory and in a browser go to:
http://localhost:3000/begin/[your drive letter]-[your directory path]
For example:
http://localhost:3000/begin/F-iTunes-Media-Music
/\ this is the url request I sent to clean up my iTunes library.
I would also recommend you use CCleaner or some other software to check for leftover duplicates which duplicate-dumper didn't find.
Anyways, maybe I'll remake this in c++ or something at a later date so others can use it but for now this is it.
